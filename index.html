CSS

Display

Existem 'n' displays

Display grid: é perfeito para criar layouts mais complexos, como a página de um jornal, onde precisa-se de várias linhas e colunas

Display Flex: Torna o elemento um flex container automaticamente transformando todos os seus filhos diretos em flex itens.

Display FlexBox:é um módulo completo e não uma única propriedade; algumas delas devem ser declaradas no container (o elemento-pai, que chamamos de flex container), enquanto outras devem ser declaradas nos elementos-filhos (os flex itens), ele lida com o layout em uma dimensão de cada vez - seja uma linha ou uma coluna.

Padding x Margin

Padding:

É o espaço entre o elemento e a borda

Margin:

É o espaço entre a borda e outro elemento

Outros

Apenas 1 arquivo de CSS (na boa prática)

Fonte- padrão: Segoiu

Color: vai ser a cor dos elementos (tags)

Backgroundcolor: vai ser a cor de fundo

Git

Git X Git-Hub

Git

É uma linguagem que manipula arquivos (é local)

Git-Hub

Ele é web, serve para conectar os dois

Repositório

É uma pasta onde se pode guardar arquivos (controle de versionamento)

Clone repositório:

Depois que mandar o repositório para a nuvem, pode excluir, com isso, ta valendo o que ta na nuvem

P.R: requisição para mudança

Os dois mexem e o melhor fica (branch), essa escolha é o dono da nuvem que faz

Mudar as duas: MERGE

Comandos

Git-Bash= terminal do Git

Push= Quando eu quero mandar coisa para a nuvem

Pull= Pegar as coisas da nuvem para mim

Commit push = comitou, e entregou para as pessoas (envia pro git)

Commit = fez e só comitou, não enviou

Sinc = faz o pull e depois o push
